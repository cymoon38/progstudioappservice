Firebase Storage 보안 규칙 설정 가이드
==========================================

⚠️ 중요: Firebase Storage 업로드를 위해 반드시 보안 규칙을 설정해야 합니다!

Firebase Console에서 다음 보안 규칙을 설정하세요:

1. Firebase Console 접속: https://console.firebase.google.com
2. 프로젝트 선택: community-b19fb
3. 좌측 메뉴에서 "Storage" 선택
4. "규칙" 탭 클릭
5. 아래 규칙을 복사하여 붙여넣기:

--- 권장 규칙 (읽기는 모두 허용, 쓰기는 인증된 사용자만) ---
rules_version = '2';

service firebase.storage {
  match /b/{bucket}/o {
    // 게시물 이미지 - 모든 사용자가 읽기 가능, 인증된 사용자만 업로드/삭제 가능
    match /posts/{userId}/{allPaths=**} {
      // 읽기는 모든 사용자 허용 (로그인하지 않은 사용자도 작품 볼 수 있음)
      allow read: if true;
      // 쓰기는 인증된 사용자이고 자신의 폴더에만 허용
      allow write: if request.auth != null && request.auth.uid == userId;
      allow delete: if request.auth != null && request.auth.uid == userId;
    }
    
    // 따라그리기 이미지 - 모든 사용자가 읽기 가능
    match /recreations/{postId}/{userId}/{allPaths=**} {
      // 읽기는 모든 사용자 허용
      allow read: if true;
      // 쓰기는 인증된 사용자이고 자신의 폴더에만 허용
      allow write: if request.auth != null && request.auth.uid == userId;
      allow delete: if request.auth != null && request.auth.uid == userId;
    }
    
    // 기타 모든 파일
    match /{allPaths=**} {
      // 읽기는 모든 사용자 허용
      allow read: if true;
      // 쓰기는 인증된 사용자만 허용
      allow write: if request.auth != null;
      allow delete: if request.auth != null;
    }
  }
}

6. "게시" 버튼 클릭하여 규칙 저장

⚠️ 주의사항:
- 위 규칙은 로그인하지 않은 사용자도 작품을 볼 수 있도록 설정되었습니다.
- 업로드, 삭제는 로그인한 사용자만 가능합니다.
- 규칙을 변경한 후 바로 적용되지만, 최대 1-2분 정도 걸릴 수 있습니다.
- 규칙 저장 후 브라우저를 새로고침하고 다시 테스트해보세요.

추가로 Firestore 보안 규칙도 확인하세요:
- Firestore > 규칙 탭에서 읽기는 모든 사용자 허용
- 쓰기/수정/삭제는 인증된 사용자만 허용하도록 설정 권장
