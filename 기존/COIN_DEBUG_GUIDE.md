# 코인 지급 디버깅 가이드

## 🔍 코인 지급 확인 방법

### 1. 브라우저 콘솔 확인

좋아요를 눌러 인기작품이 선정되면 다음 로그가 순서대로 표시되어야 합니다:

#### 정상적인 로그 순서:
```
🔍 좋아요 상태 확인: {wasLiked: false, likesBeforeUpdate: 1, isBecomingPopular: true, ...}
🔍 인기작품 선정 체크 시작...
📊 게시물 상태 확인: {currentLikes: 2, isPopular: false, ...}
🎉 인기작품 선정! 코인 지급 시작...
✅ isPopular 플래그 설정 완료
💰 좋아요 누른 사용자들에게 코인 지급 시작...
💰 addCoins 함수 호출: {userId: "...", amount: 5, ...}
📝 사용자 문서 조회 시작: ...
💰 코인 업데이트: {currentCoins: 0, amount: 5, newCoins: 5}
✅ 코인 업데이트 완료
📝 코인 내역 추가 시작: ...
✅ 코인 내역 추가 완료
✅ 코인 지급 완료: ...
💰 글쓴이에게 코인 지급 시작...
💰 글쓴이 코인 지급 시작: ..., 10코인
✅ 글쓴이 코인 지급 완료: ...
✅ 인기작품 코인 지급 완료
```

### 2. 문제 확인 체크리스트

#### ❌ 코인 지급이 안 되는 경우:

**A. "🔍 인기작품 선정 체크 시작..." 로그가 없음**
- 원인: `isBecomingPopular` 조건이 false
- 확인: 좋아요가 정확히 1개에서 2개로 증가했는지 확인
- 해결: 좋아요가 1개인 게시물에 좋아요를 눌러야 함

**B. "📊 게시물 상태 확인"에서 `isPopular: true`**
- 원인: 이미 인기작품으로 선정됨
- 확인: 게시물이 이미 인기작품인지 확인
- 해결: 아직 인기작품이 아닌 게시물로 테스트

**C. "❌ 코인 지급 오류" 메시지**
- 원인: Firestore 보안 규칙 또는 권한 문제
- 확인: 오류 메시지 내용 확인
- 해결: `firestore.rules` 파일 확인 및 적용

**D. "⚠️ UID를 찾을 수 없음" 메시지**
- 원인: 사용자명으로 UID를 찾지 못함
- 확인: `users` 컬렉션에 해당 사용자 문서가 있는지 확인
- 해결: 사용자 문서가 정상적으로 생성되었는지 확인

### 3. 코인 잔액 확인

#### 방법 1: 코인 모달 확인
1. 상단 헤더의 코인 아이콘 클릭
2. 코인 모달에서 잔액 확인
3. "내역" 탭에서 코인 내역 확인

#### 방법 2: Firebase Console 확인
1. Firebase Console > Firestore Database
2. `users` 컬렉션에서 사용자 문서 확인
3. `coins` 필드 값 확인
4. `coinHistory` 컬렉션에서 내역 확인

### 4. 코인 지급 지연 시간

#### 정상적인 지연 시간:
- **즉시 ~ 3초**: 정상 범위
- **3초 ~ 10초**: 네트워크 지연 가능
- **10초 이상**: 문제 가능성 높음

#### 지연 원인:
1. **Firestore 쿼리 지연**: 게시물 다시 가져오기 (0.5~2초)
2. **UID 찾기**: 사용자명으로 UID 찾기 (0.5~1초)
3. **코인 업데이트**: Firestore 업데이트 (0.5~1초)
4. **코인 내역 추가**: Firestore 추가 (0.5~1초)

### 5. 자동 코인 잔액 업데이트

코인 지급 후 자동으로 잔액이 업데이트되지 않을 수 있습니다.

#### 수동 업데이트 방법:
1. 페이지 새로고침 (F5)
2. 코인 모달 열기 (자동 업데이트)
3. 상단 헤더의 코인 아이콘 클릭

### 6. 문제 해결 단계

#### Step 1: 콘솔 로그 확인
- 좋아요 클릭 후 콘솔에 어떤 로그가 출력되는지 확인
- 오류 메시지가 있는지 확인

#### Step 2: 조건 확인
- 좋아요가 정확히 1개에서 2개로 증가했는지 확인
- 게시물이 이미 인기작품이 아닌지 확인

#### Step 3: Firestore 확인
- Firebase Console에서 `posts` 컬렉션 확인
- `isPopular: true`로 설정되었는지 확인
- `coinHistory` 컬렉션에 내역이 추가되었는지 확인
- `users` 컬렉션의 `coins` 필드가 증가했는지 확인

#### Step 4: 보안 규칙 확인
- `firestore.rules` 파일이 Firebase Console에 적용되었는지 확인
- `coinHistory` 컬렉션 생성 권한 확인
- `users` 컬렉션 `coins` 필드 업데이트 권한 확인

### 7. 자주 발생하는 문제

#### 문제 1: "Missing or insufficient permissions"
- **원인**: Firestore 보안 규칙 문제
- **해결**: `firestore.rules` 파일을 Firebase Console에 적용

#### 문제 2: "사용자 문서가 없습니다"
- **원인**: 사용자 문서가 생성되지 않음
- **해결**: 회원가입이 정상적으로 완료되었는지 확인

#### 문제 3: "UID를 찾을 수 없음"
- **원인**: 사용자명과 UID 매칭 실패
- **해결**: `users` 컬렉션에서 사용자명(`name` 필드) 확인

#### 문제 4: 코인은 지급되었지만 화면에 반영 안 됨
- **원인**: 코인 잔액 자동 업데이트 실패
- **해결**: 페이지 새로고침 또는 코인 모달 열기

### 8. 테스트 시나리오

#### 정상적인 테스트:
1. 좋아요가 1개인 게시물 찾기
2. 좋아요 클릭 (2개가 되어 인기작품 선정)
3. 콘솔 로그 확인
4. 코인 모달에서 잔액 확인
5. Firebase Console에서 `coinHistory` 확인

#### 문제 재현:
1. 이미 인기작품인 게시물에 좋아요 클릭
2. 좋아요가 0개 또는 2개 이상인 게시물에 좋아요 클릭
3. 로그인하지 않은 상태에서 좋아요 클릭
